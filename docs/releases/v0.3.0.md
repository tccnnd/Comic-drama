# Release v0.3.0 — global-consistency-governance

Date: 2026-06-06
Branch: TBD (local, uncommitted at time of writing)

## Summary

Adds a governance layer over the existing continuity machinery. It scores five
continuity dimensions (character, lighting, environment, prop, camera) per
scene, aggregates them into a structured verdict, rolls verdicts into a
project-level ledger, and enforces a `report`/`block` policy. The
`consistency_validator` stays a stateless scoring engine; all policy, state,
and aggregation live in the new orchestrator. No governance-driven regeneration
in this release.

## Highlights

- **Five-dimension scoring**: `validate_prop_continuity` and
  `evaluate_camera_continuity` added to `consistency_validator.py` (stateless),
  alongside the existing character/style/lighting checks.
- **Props registry**: recurring objects in the production bible with optional
  reference image, owning characters, and scenes.
- **Governance orchestrator** (`backend/consistency_governance.py`):
  - `evaluate_scene_governance` → per-scene verdict (§ verdict contract), with
    fail>warn>pass precedence and info-neutral missing references.
  - `apply_governance_policy` → `report` | `block` (default `report` via
    `CONSISTENCY_POLICY_MODE`).
  - `build_continuity_ledger` → project rollup (status counts, per-dimension
    pass rates, offending scenes, blocked count).
- **Persistence + surface**: verdict stored at `scene["governance"]` after
  render; snapshot exposes `continuity_ledger`; review console shows continuity
  chip, per-scene badges, and five-dimension detail.
- **Enforcement**: `block`-mode + not-deliverable scenes are flagged in export
  readiness.

## Architecture note

`consistency_validator.py` remains a stateless scoring engine (image-in /
score-out). The orchestrator owns verdict aggregation, policy, persistence, and
the ledger. This separation keeps the validator reusable and bounds the blast
radius of governance changes.

## Known limitations

- **Browser visual smoke unverified**: the in-app browser blocks
  localhost/127.0.0.1 with `ERR_BLOCKED_BY_CLIENT`, so the review-console
  continuity display was not visually smoke-tested. JS syntax is validated by
  `node --check`; backend logic by tests. Re-run a visual check when a usable
  browser environment is available.
- **No regeneration**: governance does not trigger re-renders in v0.3.0. The
  `regenerate` policy mode is deferred to a later spec to avoid a render
  feedback loop. `CONSISTENCY_MAX_RETRIES` stays validator/runtime config and is
  non-operative for governance policy.
- **Similarity method**: histogram/structural-hash scoring; embedding-based
  identity/motion analysis is a future upgrade.
