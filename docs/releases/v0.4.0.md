# Release v0.4.0 — director-review-console

Date: 2026-06-06
Branch: `codex/director-review-console-impl` (local, based on the v0.3.0 branch)

## Summary

Evolves the storyboard review canvas, in place, into a production
review-and-rerender console. It does not add new generation, governance, or
provider logic and introduces no new persisted schema. It consumes data already
in the project snapshot — review state, v0.2.0 provenance (`generation_meta`),
v0.3.0 governance (`scene["governance"]`), and the `continuity_ledger` — and
makes it unified, triageable, and actionable (per-scene and serial batch
rerenders via existing endpoints).

## Highlights

- **Overview header**: continuity ledger counts, real-vs-fallback provenance
  counts, review progress; metrics drive the triage filter.
- **Triage**: filter by review status, governance status, provenance,
  deliverability, rating; sort by order / rating / governance severity /
  fallback-first. Client-side, live-derived.
- **Unified review unit**: reuses the existing v0.2.0 generation and v0.3.0
  governance badges/detail with review state and asset readiness.
- **Rerender actions**: per-scene (image/audio/video/full) and a serial batch
  rerender over the filtered set with confirmation, progress, per-scene
  outcomes, and fail-isolation — existing endpoints only.

## Implementation slices

- Slice A (read-only console, commit `d088984`): overview, triage, unified
  review unit, in-place review edits, pure helpers + tests.
- Slice B + wrap-up (commit `576851c`): per-scene and serial batch rerender,
  console docs.

## Architecture / boundary notes

- Frontend-first: `frontend/utils.js`, `state.js`, `render.js`, `events.js`,
  `api.js`, `styles.css`. No backend change was needed (all overview counts are
  client-derivable from the snapshot).
- No new generation/governance/provider logic; no new endpoints; no new
  persisted schema. Overview and triage are live-derived/client-side.
- Reuses v0.2.0/v0.3.0 display components rather than reimplementing provenance
  or governance rendering.

## Validation

  `state.js`, `utils.js` — pass.
- `node tests/test_review_console_helpers.mjs` — passed (overview counts;

  unknown/not_evaluated).

## Known limitations

- **Browser visual smoke unverified**: the in-app browser blocks
  localhost/127.0.0.1 with `ERR_BLOCKED_BY_CLIENT`, so the console was not
  visually smoke-tested. Syntax validated by `node --check`; helper logic by
  tests. Re-run a visual check in a browser that allows localhost.
- **Serial batch by design**: batch rerender runs sequentially (no concurrency)
  and is reviewer-initiated. Governance-driven automatic rerender (`regenerate`)
  remains deferred to a future spec.
- **Baseline dependency**: the console builds on v0.2.0 provenance and v0.3.0
  governance; without them on the baseline, scenes show unknown/not_evaluated.
