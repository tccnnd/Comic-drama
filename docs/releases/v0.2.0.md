# Release v0.2.0 — video-provider-mainline

Date: 2026-06-06
Branch: `codex/video-provider-mainline`

## Summary

Makes real video generation the primary scene rendering path, with local 2.5D
as an explicit, observable fallback. Adds a first-class `shot_plan`, per-scene
generation provenance, canonical-timeline enrichment, a unified fallback
policy, and review-console visibility into how each scene was produced.

## Highlights

- **shot_plan**: persisted per scene; derived from `temporal_spec.shots` or
  synthesized as a single full-duration shot; contiguous and covering the scene
  duration. `build_canonical_timeline` consumes the same derivation so the
  timeline and the persisted plan stay identical.
- **Generation provenance** (`generation_meta`): `provider_id`,
  `provider_label`, `backend`, `is_real_video`, `fallback_used`, `attempts`,
  `duration_seconds`, `fallback_mode`, sanitized `error`, `generated_at`.
- **Canonical timeline**: per-clip `metadata.generation` + `shot_plan_source`;
  project `summary` adds `real_video_scene_count` / `fallback_scene_count`.
- **Unified fallback policy**: `VIDEO_FALLBACK_MODE=report|strict|silent` with
  `VIDEO_STRICT` override; default `report`.
- **Review console**: per-scene real-video / fallback / unknown badge with
  provider label, attempts, and failure reason; project-level provider status.

## AC-7 acceptance — qualified pass

The sample workflow was run end-to-end with a local keyframe provider to bypass
the environment-blocked ComfyUI keyframe tunnel:

```powershell
python -m scripts.run_workflow --input inputs\sample_story.txt --keyframe-provider local
```

Result: completed end-to-end, producing a final video and
`canonical_timeline.json`. The run validated the mainline live — accurate
`real_video_scene_count` / `fallback_scene_count`, populated `shot_plan_source`,
and full per-clip `generation` provenance — and exercised the report-mode
fallback for real: the remote provider returned HTTP 429, retried per
`VIDEO_MAX_RETRIES`, then fell back to 2.5D with provenance recorded (sanitized
error, no credentials leaked).

Caveats:
- Live run validated the **report-mode fallback** branch. The **real-video
  success** branch is covered by mock tests; a live provider success run is
  still quota-dependent (the remote provider was rate-limited during this run).
- The **ComfyUI keyframe tunnel** remains environment-blocked
  (`Error reading SSH protocol banner`) and is upstream of the video-provider
  path this release changes.
