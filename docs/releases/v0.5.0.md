## Summary

Makes the AI director's interpretation a first-class, structured stage between
scene classification and video-provider prompt construction. Adds a per-scene
`director_plan` (the "why"), a structured per-shot `visual_content` (the "what
is on screen"), and a deterministic `visual_prototype` shot-language constraint
layer. The video-provider prompt now consumes `visual_content` as its primary
visual driver instead of raw dialogue. Deterministic-first: no LLM and no new
provider calls.

Acceptance chain:

```text
script â†’ director_meta â†’ director_plan â†’ shot_plan + visual_content
       â†’ video provider prompt
```

## Highlights

- **director_plan** (`build_director_plan`): `dramatic_intent`,
  `emotional_target`, `narrative_focus`, `rationale`, `source`
  (`rules`/`default`), synthesized from `director_meta` + scene text.
- **visual_content** (`build_shot_visual_content`): 8 structured fields per shot
  + `_source` provenance (`prototype` / `rules` / `legacy`), `shot_size`,
  `camera_language` (lens/dof/movement/framing), and `dramatic_intent`.
- **visual_prototype**: deterministic shot-language constraint layer
  (`prototype_lock` | `freeform`, fixed id set, hard/soft/guideline constraints,
  gap reason).
- **Prompt consumption**: `build_scene_video_prompts` builds primarily from
  `visual_content` + constraints; dialogue demoted to context; legacy fallback
  retained.
- **Pipeline + persistence**: `build_shot_plan` enriches shots additively;
  workflow persists `director_plan` + enriched `shot_plan`; legacy projects
  normalized on load/snapshot.

## Scope note

Slice C was implemented broader than the original "consume visual_content"
wording: it introduces the deterministic `visual_prototype` constraint layer.
This is recorded in `.kiro/specs/director-interpretation-mainline/tasks.md`. The
layer is deterministic and consistent with the spec's deterministic-first
philosophy, and is a natural seam for the planned
`provider-quality-feedback-loop` (quality scoring can later check generated
output against these constraints).
